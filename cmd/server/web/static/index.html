监听与 TLS

            
UDP/TCP 为传统 DNS 端口；DoH 为 DNS over HTTPS；最后一项为管理后台与 REST API 监听地址。

            

              
              
              
              
              
            

            
DoH TLS 使用下方 PEM：可粘贴全文，或上传 .pem / .crt / .key 文件填入文本框。保存后需重启 DoH 监听生效。若仅在配置文件中填写 listen.tls_cert / tls_key 路径，控制台不会显示该路径，保存时也不会被清除。

            
            
          

          

            
DNS 应答

            
开启后对所有客户端生效：不再向外返回 IPv6 地址记录。

            
            
AAAA 查询仅返回无应答的 NOERROR（NODATA）；A、CNAME 等应答中的 AAAA 记录会被移除。保存后立即生效。

解析器

            
控制写入缓存的最长 TTL、上游超时与重试；「非白名单」指未命中白名单的域名。

            

              
              
              
              
            

          

          

            
客户端限速

            
按客户端 IP（或 DoH 推导的标识）限制 QPS；突发为令牌桶允许的短时间峰值。

            

              
              
            

          

          

            
缓存预热

            
进程启动后异步预解析下列域名并写入 Redis。记录类型用英文逗号分隔，如 A、AAAA。

            
            
          

          

            
统计持久化（Redis）

            
将仪表盘计数定时写入 Redis；间隔为 0 则关闭定时写入（启动时仍会尝试从 Redis 恢复）。变更后建议保存并重启以稳定周期。

            

              
              
            

          

        


        

          

            
VIP → 真实 IP 映射

            
将 VPN 虚拟 IP 映射为用于 ECS 的公网 IP。可选 HTTP 接口；映射结果会缓存到 Redis，有效期见下方 TTL。若客户端已是公网 IP，将直接使用，不查映射。

            
映射 API 留空时：Redis 未命中后使用本机公网出口 IP（由下方探测 URL 获取，默认 ipify）写入 Redis。

            
默认 ECS：发往国内上游时，若 VIP 无法解析出映射/公网 IP，则用「默认国内 ECS」作为 EDNS Client Subnet；发往海外上游时，在客户端未带 ECS 且映射 IP 也不可用时，用「默认全球 ECS」（仍优先使用客户端 ECS 与映射得到的 IP）。

查询日志持久化

            
max_entries > 0 时写入 Redis 列表（重启不丢，约保留最近 N 条）；0 时仅存进程内约 500 条，重启清空。保存配置后生效。

            

              
              
            

          

          

            
查询日志

            
时间、请求（域名 + 类型）、响应（记录摘要与路由）、客户端（源地址 / 映射 IP / ECS）。每页内最新记录在上方；支持分页拉取，减轻大量日志时的卡顿。可按域名、源 IP、映射 IP 或关键词筛选（与关系）。一键清空会删除当前后端中的全部查询日志（Redis 列表或进程内缓冲），不可撤销。

监听与 TLS

            
UDP/TCP 为传统 DNS 端口；DoH 为 DNS over HTTPS；最后一项为管理后台与 REST API 监听地址。

            

              
              
              
              
              
            

            
DoH TLS 使用下方 PEM：可粘贴全文，或上传 .pem / .crt / .key 文件填入文本框。保存后需重启 DoH 监听生效。若仅在配置文件中填写 listen.tls_cert / tls_key 路径，控制台不会显示该路径，保存时也不会被清除。

            
            
          

          

            
DNS 应答

            
开启后对所有客户端生效：不再向外返回 IPv6 地址记录。

            
            
AAAA 查询仅返回无应答的 NOERROR（NODATA）；A、CNAME 等应答中的 AAAA 记录会被移除。保存后立即生效。

国内上游（CN）

            
填 HTTPS URL 时，用「DoH 方式」选择 RFC 8484（POST 二进制）或 JSON GET（/resolve）；选自动时按 URL 路径判断（以 /resolve 结尾则用 JSON）。纯 UDP 只填「UDP 地址」。权重越大越常被选中。

解析器

            
控制写入缓存的最长 TTL、上游超时与重试；「非白名单」指未命中白名单的域名。

            

              
              
              
              
            

          

          

            
客户端限速

            
按客户端 IP（或 DoH 推导的标识）限制 QPS；突发为令牌桶允许的短时间峰值。

            

              
              
            

          

          

            
缓存预热

            
进程启动后异步预解析下列域名并写入 Redis。记录类型用英文逗号分隔，如 A、AAAA。

            
            
          

          

            
统计持久化（Redis）

            
将仪表盘计数定时写入 Redis；间隔为 0 则关闭定时写入（启动时仍会尝试从 Redis 恢复）。变更后建议保存并重启以稳定周期。

            

              
              
            

          

        


        

          

            
VIP → 真实 IP 映射

            
将 VPN 虚拟 IP 映射为用于 ECS 的公网 IP。可选 HTTP 接口；映射结果会缓存到 Redis，有效期见下方 TTL。若客户端已是公网 IP，将直接使用，不查映射。

            

              客户端 VIP
              →
              映射 API（或 Redis）
              →
              公网 IP（ECS 源）
              →
              GeoIP 分流
              →
              CN / OUT 上游
              →
              DNS 应答
            

            
上图仅为逻辑示意；实际以配置与解析路径为准。缓存键形态：dns:域名:类型:global 或 dns:域名:类型:ecs:子网。

            
映射 API 留空时：不再请求外呼映射，也不会用本机公网探测去写 vip:*（避免误占国内 ECS 的 default_cn_ecs 保底）；仅 Redis 命中或回退为解析 VIP 字面 IP。需要外呼时请填写 GET 接口（查询参数 ip）。下方「本机公网探测 URL」保留字段，当前不用于上述 VIP 回填。

            
默认 ECS：国内上游 ECS 优先级为——① 客户端公章网 EDNS 子网；② VIP→realIP 映射公网（Redis / API）；③ mapper.default_cn_ecs 保底。海外路径见配置说明。

域名黑名单

            
每行一个域名（与 DNS 查询名一致、小写）。命中后应答与解析 → 非白名单域名策略一致（NXDOMAIN 或解析到本机回环）。保存后写入 security.blacklist。

清理 DNS 缓存

            
删除上方模式中所有匹配键（与列表使用同一通配符）。默认 dns:*。

            

              
            

          

        


        

          

            
查询日志持久化

            
max_entries > 0 时写入 Redis 列表（重启不丢，约保留最近 N 条）；0 时仅存进程内约 500 条，重启清空。保存配置后生效。

            

              
              
            

          

          

            
查询日志

            
时间、请求（域名 + 类型）、响应（记录摘要与路由）、客户端（源地址 / 映射 IP / ECS）。点击详细可查看是否走国内/海外上游、是否携带 ECS、过程步骤及响应文本。每页内最新记录在上方；支持分页与筛选。一键清空会删除当前后端中的全部查询日志，不可撤销。

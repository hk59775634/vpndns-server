# 高并发系统优化说明

本文档说明 **操作系统与中间件层面** 的调优建议，与仓库内 Go 代码中的高并发改动（异步查询日志、连接池、UDP 缓冲等）配合使用。修改内核参数前请在测试环境验证，并保存可回滚的配置。

---

## 操作系统初始化脚本（推荐）

仓库提供 **`scripts/init-highconcurrency-os.sh`**，用于把本文中与 **Linux 内核 sysctl**、**全局限额 `fs.file-max`**、**PAM `limits.d` nofile**、**systemd `DefaultLimitNOFILE`** 及（在 `nf_conntrack` 已加载时）**`nf_conntrack_max`** 相关的建议 **写成持久配置并按需刷入运行时**。脚本可 **重复执行**：会先比对磁盘上的目标文件与当前内核参数，**内容未变则不重复写入、不重复备份**；**不会**在未加参数的情况下覆盖「已存在但无本脚本管理标记」的同名配置文件，避免与人工或其它工具写的 `sysctl.d` / `limits.d` 混用。

**必须以 root 执行**，在仓库根目录或任意路径下均可（以下示例假定当前目录为仓库根）。脚本为 **Bash** 编写；若误用 `sh scripts/...`（在 Ubuntu 上 `sh` 多为 dash），脚本会在开头 **自动改用 bash 再执行**，无需改命令。

```bash
sudo ./scripts/init-highconcurrency-os.sh
```

**常用参数**（可组合使用，例如 `--sysctl-only --apply-runtime`）：

| 参数 | 含义 |
|------|------|
| （无） | 完整流程：写入/校验持久文件；若本次改写了 sysctl drop-in 或你显式要求刷运行时，则对 **本脚本管理的** `sysctl.d` 执行 `sysctl -p`；若 systemd drop-in 有变更则 `daemon-reload`。 |
| `--sysctl-only` | 只处理 sysctl（及 conntrack drop-in），不修改 `limits.d` 与 systemd。 |
| `--apply-runtime` | 在 **不修改** 持久文件的前提下，仍对磁盘上已存在的本脚本 sysctl 文件执行 `sysctl -p`（适用于：文件已正确但运行时尚未同步）。 |
| `--takeover` | 当目标路径已存在且 **不含** 脚本内标记行时，默认会 **中止**；加此参数则先备份为 `*.foreign.bak.时间戳` 再覆盖为本脚本内容。 |
| `-h` / `--help` | 打印简要用法说明。 |

**脚本写入的标记**：持久文件首行附近含 `# managed-by: vpndns init-highconcurrency-os.sh`，用于识别「由本脚本管理」。若你曾手工编辑过这些文件，建议保留或补上该标记，或改用 `--takeover` 并事先自行备份。

**与下文 §2 的关系**：§2 中的 `sysctl` 示例与脚本生成的主配置文件一致思路；日常运维 **优先用脚本落盘**，无需再手动执行全量 `sysctl --system`（脚本仅对 **本仓库定义的两个** `sysctl.d` 文件做 `sysctl -p`）。若需回滚，可使用同目录下脚本在变更前自动生成的 `*.bak.*` 备份，或删除对应 drop-in 后按发行版方式重载 sysctl。

## 1. 文件描述符与进程限制

DNS/DoH 高并发下，每个 UDP 套接字、TCP 连接、上游 HTTP 连接、Redis 连接都会占用文件描述符。

- **进程 `nofile`（硬/软限制）**  
  - 临时：`ulimit -n 1048576`  
  - 持久：systemd 服务中已示例 `LimitNOFILE=1048576`（见 `deploy/vpndns-server.service`），按需增大。**安装、启用、`systemctl edit` 覆盖与停用**等步骤见 **[docs/DEPLOY.md](docs/DEPLOY.md)** 的 **「4. systemd」** 一节。
- **全局限额**（Linux）：`/proc/sys/fs/file-max` 过低时，即使 `ulimit` 够大也会失败；可检查 `cat /proc/sys/fs/file-max`，必要时在 `sysctl.d` 中提高（需与运维规范一致）。

## 2. 网络内核参数（Linux sysctl）

以下项对 **大量 UDP DNS** 与 **大量短连接/HTTP/2** 有帮助。数值需按机器内存与业务峰值调整，**不是越大越好**。

在 `/etc/sysctl.d/99-vpndns-highconcurrency.conf` 中示例（请按环境删减）：

```conf
# 套接字接收/发送缓冲上限（需 >= 应用 setsockopt 的值；本服务 UDP 尝试设到 4MiB 级别）
net.core.rmem_max = 8388608
net.core.wmem_max = 8388608
net.core.rmem_default = 262144
net.core.wmem_default = 262144

# 网卡入队 backlog，突发流量时减少内核丢包
net.core.netdev_max_backlog = 16384

# SYN 队列与已建立连接队列（主要影响 TCP DoH/管理端口）
net.core.somaxconn = 4096
net.ipv4.tcp_max_syn_backlog = 8192

# TIME_WAIT 套接字复用（有大量短连接时可考虑，注意与负载均衡策略兼容）
net.ipv4.tcp_tw_reuse = 1

# 本地端口范围（若本机作为客户端大量访问上游 DoH）
net.ipv4.ip_local_port_range = 1024 65535
```

应用：可执行上一节 **`scripts/init-highconcurrency-os.sh`** 持久落盘并按需 `sysctl -p`；或手动 `sudo sysctl --system` / `sudo sysctl -p /etc/sysctl.d/99-vpndns-highconcurrency.conf`。

**说明**：`tcp_tw_reuse` 等行为随内核版本与场景而异，生产变更前请查阅当前发行版文档。

## 3. 防火墙与连接跟踪（nf_conntrack）

若启用 **iptables/nftables + conntrack**，极高 QPS 下可能打满连接跟踪表，表现为随机丢包、超时。

- 检查：`conntrack -C`、`dmesg | grep nf_conntrack`  
- 可调（示例）：`net.netfilter.nf_conntrack_max`、`nf_conntrack_buckets`（路径因内核而异）。  
- 若 DNS 仅在内网可信区，部分环境会对 **53/UDP** 不做跟踪或单独限流，需由安全策略决定。

## 4. Redis 服务端

客户端已在程序侧增大连接池（`PoolSize` / `MinIdleConns`）；Redis **服务端**仍需匹配：

- `maxclients`：大于业务峰值连接数。  
- `tcp-backlog`：与内核 `somaxconn` 配合。  
- 持久化（AOF/RDB）在磁盘慢时会造成延迟尖刺，高 SLA 场景可评估 **独立实例** 或 **降低持久化频率**（权衡数据安全）。

---

## 5. CPU、NUMA 与中断

- **多队列网卡（RSS/RPS）**：高 PPS 时让软中断分布到多核，避免单 CPU 打满。  
- **绑核**：可将 `vpndns-server` 与 Redis、监控进程分离到不同 NUMA 节点，减少跨节点内存访问（需实测）。  
- **节能策略**：高性能场景可考虑 `cpufreq` 性能模式（增加功耗）。

---

## 6. 监控与排障

- **`GET /api/v1/meta`** 中 **`query_log_async_dropped`**：异步查询日志队列满时丢弃条数；持续非零说明写入速度跟不上，应增大 `query_log` 批量落库能力或降低日志量。  
- **Prometheus `/metrics`**：结合 `rate_limited_total`、`resolve_errors_total`、Redis/上游延迟。  
- **`ss -u -a` / `dropwatch` / `nstat`**：辅助判断内核侧 UDP 丢包。

## 7. 与代码改动的对应关系

| 方向 | 代码侧（摘要） | 系统侧建议 |
|------|----------------|------------|
| 查询日志 | 异步队列 + 批量写 Redis / 内存，避免每条查询同步 Pipeline | Redis `maxclients`、磁盘与网络 |
| 上游 DoH | 扩大 `http.Transport` 空闲连接与每主机上限 | 本地 ephemeral 端口、`tcp_tw_reuse` 等 |
| Redis 访问 | 增大 go-redis 连接池 | `maxclients`、`somaxconn` |
| UDP DNS | `SO_RCVBUF`/`SO_SNDBUF` + `UDPSize` | `rmem_max` / `wmem_max`、`netdev_max_backlog` |
| DoH HTTP | `ReadHeaderTimeout` / `IdleTimeout` | `somaxconn`、负载均衡器空闲超时与后端一致 |
